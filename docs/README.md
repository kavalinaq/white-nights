# WHITE NIGHTS — Documentation

Themed social network for readers: short literary reviews, book impressions, quotes, shelves, reading tracker.

## How to read these docs

1. Start with [overview.md](overview.md) for vision, scope, personas, tech stack.
2. Read [architecture.md](architecture.md) for system layout and open decisions.
3. Read [data-model.md](data-model.md) for the DB schema and known gaps.
4. Read [roadmap.md](roadmap.md) for the phased build plan.
5. Each [feature file](features/) is self-contained: priority, complexity, dependencies, user stories, functional requirements, API sketch, acceptance criteria.
